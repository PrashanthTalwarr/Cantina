# Draft Outreach: Personalized Messages to Real Targets

## Outreach Philosophy

Every message follows: **Signal** (I noticed something specific) → **Data** (third-party evidence of why it matters) → **Bridge** (how Cantina maps to this) → **Soft ask** (15 min, specific value offered).

Category-specific risk hooks make each message feel researched, not templated:
- **DEX** → MEV extraction, reentrancy in swap logic
- **Lending** → oracle manipulation, liquidation edge cases
- **Yield/Vault** → accounting errors, withdrawal race conditions
- **Bridge** → cross-chain message validation, replay attacks
- **Stablecoin** → depeg scenarios, minting function access control
- **L2/Infra** → sequencer downtime, fraud proof edge cases

In production, these are generated by Claude API using the enrichment data. Below are 3 examples showing what "good" looks like.

## How AI Scales This

In production, the outreach agent (Claude API + LangChain) generates these automatically:

1. **Input:** Enrichment data from the pipeline (TVL, audit status, category, velocity, persona)
2. **System prompt:** Cantina positioning, category-specific risk hooks, writing rules
3. **Output:** Personalized message + subject line + signals used (for tracking)

The human reviews and approves before sending. The LLM does the heavy lifting on personalization; the human ensures quality and catches hallucinations. This is how you scale from 5 outreach messages to 50/week without losing personalization quality.
